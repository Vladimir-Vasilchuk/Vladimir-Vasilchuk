# Спецификация для веб-приложения

## Описание проекта

В текущем проекте представлен комплексный артефакт, необходимый для дальнейшего этапа разработки - проектирования базы данных и реализации функционала приложения. Он закладывает основу для стабильной и эффективной работы веб‑сервиса, ориентированного на рост продаж в высокий сезон.

Цель проекта:
* Разработать обновлённую версию спецификации требований к веб‑приложению «Накарабине» для продажи товаров для активного отдыха.
* Создать логическую модель данных в нотации Crow’s Foot и привести её к третьей нормальной форме (3НФ)
* Сформировать словарь данных с детальным описанием сущностей, их атрибутов
* Интегрировать логическую модель и словарь данных в спецификацию требований - как основу для проектирования базы данных и дальнейшей разработки веб‑приложения

Этапы проектирования:
* Построение контекстной и Use Case‑диаграммы
* Формулирование функциональных и нефункциональных требований
* Проектирование логической модели данных (3НФ)
* Создание словаря данных
* Описание атрибутов качества
* Формулирование бизнес‑правил и составление глоссария
